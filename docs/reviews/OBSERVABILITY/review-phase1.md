## Findings

### 1. [nit / minor] Prometheus datasource has no stable `uid`
`grafana/provisioning/datasources/prometheus.yml`

The datasource omits `uid:`. Grafana will synthesise one from the name, which is stable enough day-to-day, but dashboard JSON in Phase 6 will need to reference it via a generated string. Pinning a deterministic UID now means dashboard JSON can hardcode it and survive any future rename.

Fix:
```yaml
apiVersion: 1
datasources:
  - name: Prometheus
    type: prometheus
    uid: prometheus
    access: proxy
    url: http://prometheus:9090
    isDefault: false
```

The existing `loki.yml` doesn't pin a uid either, but Phase 6 dashboards are Prometheus-fronted, so this one matters now.

### 2. [nit] Scrape `scrape_interval` mismatch between `global` and the job
`prometheus/prometheus.yml:11-18`

`global.scrape_interval: 15s` is overridden to `10s` for the single job. Harmless and intentional per plan, but the `global` value is now dead config — nothing else will ever scrape. Either drop the global override or align the global value to `10s`. Mild readability win, no functional impact.

### 3. [nit] `prometheus` container runs as root
`docker-compose.yml:71-83`

The `bot-manager` service uses `user: "${HOST_UID}:${HOST_GID}"` so bind-mounted `./logs` is writable by the host user without chown. The new `prometheus` service has no `user:` directive and the TSDB lives in a named volume (not bind-mounted), so this is fine for now. Flagging it because the `./prometheus/prometheus.yml` bind-mount is ro and config edits happen from the host — no permission interaction. If Phase ≥2 ever needs Prometheus to write to a host-mounted directory (e.g., rule files generated by the app), revisit.

### 4. [advisory] Consider `management.endpoint.prometheus.access=read-only` for Spring Boot 3.4
`src/main/resources/application.properties:34-37`

Boot 3.4 introduced fine-grained per-endpoint access control. Defaults are fine and the endpoint is already gated by exposure, so this is purely informational. No change required.
